Restash — Onboarding flow

Four hands-on steps. Each one advances when the user actually does the thing (no "Next" button) — except where macOS hands us off (Accessibility). The whole flow lives in the same popover the app uses every day, so the onboarding is the app teaching itself.

Design notes:
  • Each card lives inside the existing 280×348 popover — no extra window, no overlay app. The same surface that ships every day teaches itself.
  • Progress dots at the bottom give the user a sense of length (4 steps) and what's done.
  • Every step has a "Skip this step" link, plus one "Skip — I'll figure it out" on Step 1 to escape the whole tour.
  • Step 2 and Step 3 advance from real user input (hotkey press, typing), not from a "Next" button — feels more like a game, less like paperwork.
  • Step 4 polls the Accessibility state. The user grants in System Settings, comes back to Restash, and the card auto-advances. No "click to verify" button needed.
  • Brand bar + progress dots stay in the same position as the real app, so users learn where things live from the start.
