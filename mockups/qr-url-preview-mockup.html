Implementation notes

Site preview

  • Fetch the URL HTML in the main process (no CORS issues there) with a short timeout (~3 s) and a small byte cap (~256 KB).
  • Parse <title>, og:title, og:description, og:image, meta[name="description"].
  • Favicon: try /favicon.ico, then icon/apple-touch-icon from the HTML, then Google's s2/favicons?domain=… as a fallback.
  • Cache by hostname in memory for ~10 min so re-decoding the same domain is instant.
  • Render the loading skeleton immediately — fetch resolves underneath. Never blocks the popover from appearing.

Safety

Two layers:

  • Heuristics (instant, no network): HTTP-only → warn. URL shortener (bit.ly, t.co, ow.ly, tinyurl, …) → warn. Sketchy TLD (.zip, .top, .tk, .xyz subdomains) → warn. Punycode (xn--) → warn. IP literal as host → warn.
  • Google Safe Browsing API (definitive, ~100 ms): free up to 10k checks/day with an API key. Returns malware / phishing / unwanted-software flags. Cached by hostname.

Heuristics gate the warning badge. Safe Browsing gates the danger badge. If both pass → safe.

Privacy

  • Both lookups send the hostname (not the full URL) to third parties. Disclose in Settings; offer an off switch.
  • OG fetch is direct to the destination site — Restash doesn't route through any intermediary.
