My pick: Option B (mini-card) for the hard blocks — stash / item / pin limits — because it fires exactly when the user is blocked, states the limit plainly, and has a one-tap path to plans plus a "Not now". For the file-size limit you already have the inline orange warning row in the editor — just add an "Upgrade for 5 GB" link to it (a touch of Option A). So: B for blocked actions, a small A-style link on the file warning. Option C is too easy to miss for something the user needs to act on.
